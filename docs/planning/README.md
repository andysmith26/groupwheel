# Planning Documents

This directory contains all active and in-progress planning specs for Groupwheel.

## Directory Structure

### [simplify-flows/](simplify-flows/)

Five-phase plan to simplify user-facing routes and reduce the mental model for teachers. Phases are sequential.

- **Phase 1:** showToClass use case (merge create + publish session)
- **Phase 2:** Live route (merge /present + /observe into /live)
- **Phase 3:** Workspace inline generation (move group size picker, delete /start)
- **Phase 4:** Smart activity entry point
- **Phase 5:** Cleanup (delete old routes)

**Status:** Specified, not yet implemented.

### [project-simplify-again/](project-simplify-again/)

Post-MVP work packages (WP1-WP4) building on simplify-flows:

- **WP1:** Speed Path — one-tap generate & show, avoid recent groupmates
- **WP2:** Observation & Feedback Loop — persistent observation UI, timer
- **WP3:** Analytics & Decision Support — contextual analytics, scenario comparison
- **WP4:** Onboarding & First Value — demo mode, quick start flows

Includes a shared context document (`groupwheel-ux-project-context.md`) with architecture rules and dependency graph.

**Status:** Specified, not yet implemented.

### [project-blizzard/](project-blizzard/)

Current active project (branch: `project-blizzard`). Consolidated UX design document with 12 design decisions grounded in Cooper's posture framework.

**Status:** Active.
